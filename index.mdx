## Overview

TaskFlow is a lightweight task management API that allows you to create projects, manage tasks, update task statuses, and delete tasks.

This API follows a simple workflow:

1. Create a project
2. Create tasks inside the project
3. Retrieve projects tasks
4. Update task status
5. Delete tasks

## Authentication

All API requests must include a bearer token in the Authorization header.

**Example:**

```json
curl https://test-writechoice.onrender.com/projects \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Quickstart

This guide walks you through the complete TaskFlow workflow:

<Steps>
  <Step title="Create a project">
    Before you start creating tasks, you must first create a project. Send a POST request to create a new project.

    ## Request

    ```shellscript
    curl -X POST https://test-writechoice.onrender.com/projects \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Documentation Project",
        "description": "Testing the TaskFlow API"
      }'
    ```

    ## Response

    ```shellscript
    {
      "project_id": "0aac3323-0469-4440-8256-676828b61915",
      "name": "Documentation Project",
      "description": "Testing the TaskFlow API",
      "created_at": "..."
    }
    ```

    Save the returned ``project_id``. You'll need it to create, list and update tasks.
  </Step>
  <Step title="Create a task">
    Use the ``project_id`` returned from the previous request.

    ## Request

    ```shellscript
    curl -X POST https://test-writechoice.onrender.com/tasks \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "project_id": "0aac3323-0469-4440-8256-676828b61915",
        "title": "Write API overview",
        "description": "Create the introduction and quickstart sections"
      }'
    ```

    ## Response

    ```shellscript
    {
      "task_id": "5c8d7ae4-99af-4432-b683-e573300992f6",
      "status": "pending"
    }
    ```

    Save the returned ``task_id``.
  </Step>
  <Step title="Retrieve project tasks">
    Retrieve all tasks associated with your project.

    ```shellscript
    curl https://test-writechoice.onrender.com/projects/0aac3323-0469-4440-8256-676828b61915/tasks \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
  </Step>
  <Step title="Update a task">
    Update the task status as needed.

    ```shellscript
    curl -X PATCH https://test-writechoice.onrender.com/tasks/5c8d7ae4-99af-4432-b683-e573300992f6 \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "status": "in_progress"
      }'
    ```
  </Step>
  <Step title="Delete a task">
    Delete the task permanently.

    ```shellscript
    curl -X DELETE https://test-writechoice.onrender.com/tasks/5c8d7ae4-99af-4432-b683-e573300992f6 \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
  </Step>
</Steps>

## Core Concepts

### \
Projects

Projects are containers used to organize tasks. You must create a project before creating tasks. 

### Tasks

Tasks belong to projects and represent individual work items. Each task starts with a \`pending\` status by default.

### Status Values

Tasks support three status values:

| Status | Meaning |
| --- | --- |
| `pending` | Task has not started |
| `in_progress` | Task is being worked on |
| `done` | Task completed |

## \
API Reference

### \
Create Project

This endpoint creates a new project.

```shellscript
POST /projects
```

#### Request example:

```shellscript
{
  "name": "Documentation Project",
  "description": "Testing the TaskFlow API"
}
```

#### Response example:

```shellscript
{
  "project_id": "0aac3323-0469-4440-8256-676828b61915",
  "name": "Documentation Project",
  "description": "Testing the TaskFlow API",
  "created_at": "2026-05-15T12:00:00.000Z"
}
```

### \
Create Task

This endpoint creates a new task inside an existing project. 

<Warning>
  You must create a project before creating tasks
</Warning>

```shellscript
POST /tasks
```

#### Request example:

```shellscript
{
  "project_id": "0aac3323-0469-4440-8256-676828b61915",
  "title": "Write API overview",
  "description": "Create the introduction and quickstart sections"
}
```

#### Response example:

```shellscript
{
  "task_id": "5c8d7ae4-99af-4432-b683-e573300992f6",
  "project_id": "0aac3323-0469-4440-8256-676828b61915",
  "title": "Write API overview",
  "description": "Create the introduction and quickstart sections",
  "status": "pending",
  "created_at": "2026-05-15T12:19:38.905Z",
  "updated_at": "2026-05-15T12:19:38.905Z"
}
```

#### \
List Tasks

This endpoint returns all tasks associated with a specific project.

<Warning>
  If the project exists but has no tasks, the API returns an empty `tasks` array.
</Warning>

```shellscript
GET /projects/{project_id}/tasks
```

#### Request example:

```shellscript
curl https://test-writechoice.onrender.com/projects/0aac3323-0469-4440-8256-676828b61915/tasks \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Response example:

```shellscript
{
  "project_id": "0aac3323-0469-4440-8256-676828b61915",
  "total": 1,
  "tasks": [
    {
      "task_id": "5c8d7ae4-99af-4432-b683-e573300992f6",
      "project_id": "0aac3323-0469-4440-8256-676828b61915",
      "title": "Write API overview",
      "description": "Create the introduction and quickstart sections",
      "status": "pending",
      "created_at": "2026-05-15T12:19:38.905Z",
      "updated_at": "2026-05-15T12:19:38.905Z"
    }
  ]
}
```

### \
Update Task

This endpoint updates one or more fields of an existing task. You can update:

- **status**
- **title**
- **description**

```shellscript
PATCH /tasks/{task_id}
```

#### Request example:

```shellscript
{
  "status": "in_progress"
}
```

#### Response example:

```shellscript
{
  "task_id": "5c8d7ae4-99af-4432-b683-e573300992f6",
  "project_id": "0aac3323-0469-4440-8256-676828b61915",
  "title": "Write API overview",
  "description": "Create the introduction and quickstart sections",
  "status": "in_progress",
  "created_at": "2026-05-15T12:19:38.905Z",
  "updated_at": "2026-05-15T12:25:00.000Z"
}
```

### Delete Task

This endpoint permanently deletes a task.

<Danger>
  This action cannot be undone!
</Danger>

```shellscript
DELETE /tasks/{task_id}
```

#### Request example:

```shellscript
curl -X DELETE https://test-writechoice.onrender.com/tasks/5c8d7ae4-99af-4432-b683-e573300992f6 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Response example:

```shellscript
{
  "message": "Task '5c8d7ae4-99af-4432-b683-e573300992f6' was successfully deleted."
}
```

### Additional Information

#### Error Handling

The following table describes the most common API error responses and how to resolve them.

| Status | Description | Possible Solution |
| :-- | :-- | --- |
| 400 | Invalid request body | Verify that all required fields are included and properly formatted. |
| 401 | Invalid or missing API key | Ensure the `Authorization` header is included in every request. |
| 404 | Task not found | Verify that the provided `project_id` or `task_id` exists. |

#### Example Error Response

```shellscript
{
  "error": "Not Found",
  "message": "No task found with task_id 'abc123'."
}
```

#### Common Issues

#### **401 Unauthorized**

This error usually occurs when the Authorization header is missing or the API key is invalid.

**Correct format:**

```shellscript
Authorization: Bearer YOUR_API_KEY
```

#### **404 Not Found**

This error occurs when the requested project or task does not exist.

Mostly caused by invalid `project_id`, invalid `task_id `or the resource was deleted previously.

#### **400 Bad Request**

This error occurs when the request body is incomplete or contains invalid values.

**Example:**

```shellscript
{
  "status": "completed"
}
```

The allowed status values are:

- `pending`
- `in_progress`
- `done`
